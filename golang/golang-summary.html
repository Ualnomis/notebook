1 简介

1.1 起源

golang 受到很多语言的启发：

  • C：语法。
  • java：通过接口实现类继承。
  • java、C#、python： 包管理。
  • unix，Plan9，Limbo：通信序列进程（communicating sequential process，CSP），协程间通信。

1.2 优点

golang 达到以下三者的平衡：

  • 易于开发

    • 静态语言的安全性和高效性

      编译型，解释型，强类型，弱类型，静态，动态语言区别是什么？1

    • 动态语言的易开发性
    • 包管理
  • 快速编译
    • 整个程序是一个执行文件
    • 编译速度快
    • 执行速度快
  • 高效执行，通过以下技术利用分布式和多核加快执行速度。
    • 协程（goroutine），通过 channel 来在协程之间交换数据。
    • 连续栈(continuous stack)，以前是使用分段栈（segement stack）2

1.3 特性

  • 支持
    • 协程
    • 垃圾收集（gc）
    • 反射
    • 包管理
    • 通过接口实现多态
    • 与其他语言进行交互
  • 不支持
    • 函数重载，操作符重载
    • 隐式转换
    • 类继承
    • 动态加载代码
    • 动态链接库
    • 泛型
    • 异常，通过 recover 和 panic 替代
    • 断言
    • 静态变量

1.4 语言文化

简单（simple）

  • 只有 25 个关键字
  • 同一个问题只使用一两种方法来解决。
  • 通过编码规范来对代码风格进行统一。

1.5 编码规范

go 语言编码规范

1.5.1 注释

  • 使用 C 风格注释。
  • 几乎所有全局作用域的类型、常量、变量、函数和被导出的对象都应该有一个合理的注释。如果这种注释（称为文档注释）出现在函数前面，例如函数 Abcd，则要以 "Abcd…" 作为开头。godoc 会收集这些注释生成技术文档。

  • package 注释，位于 package 声明上一行，不能空行 

     // Package superman implements methods for saving the world.
 //
 // Experience has shown that a small number of procedures can prove
 // helpful when attempting to save the world.
 package superman

  • func 注释 

     // enterOrbit causes Superman to fly into low Earth orbit, a position
 // that presents several possibilities for planet salvation.
 func enterOrbit() error {
    ...
 }
  • inline 注释

2.2.1 GOPATH

GOPATH 用于解析 import 声明。GOPATH 中的路径下必须包含 bin、src、pkg 三个目录。 

mkdir ~/gospace

在~/.profile 中修改如下环境变量。 

export GOPATH="$HOME/gospace"
export PATH=$PATH:$(go env GOPATH)/bin

GOPATH 中第一个目录要固定，这样以后安装的第三方库都会安装到这个目录下，而且应该将该目录下的 bin 加入到 PATH 中。

3 工程布局3

参阅 How to Write Go Code

所有的 go 程序都应该在一个共同的 workspace 下面，这与其他语言不同，其他语言中一个项目就是一个单独的 workspace。这样组织方便 go build，go install 等工具的使用。

3.1 包

go get 命令会将项目下的所有子目录和源码文件存放到第一个工作区的 src 目录下，而 src 目录下的所有子目录都会是某个代码包导入路径的一部分或者全部。所以应该 直接在项目目录下存放子代码包和源码文件 ，并且 直接存放在项目目录下的源码文件所声明的包名应该与该项目名相同 。这样做可以让其他人使用 go get 命令从 Github 站点上下载你的项目之后直接就能使用它。

package name 应该是小写的单词，不要使用下划线或者驼峰命名法。 

go_test_lib
└── stringutils
    ├── reverse.go
    └── reverse_test.go

库代码文件 reverse.go 

package stringutils

func Reverse(s string) string {
    r := []rune(s)
    for i, j := 0, len(r)-1; i < len(r)/2; i, j = i+1, j-1 {
        r[i], r[j] = r[j], r[i]
    }
    return string(r)
}

3.2 命令程序

命令程序和库代码有所不同，每个命令程序项目应该作为一个 gopath 路径进行部署。

  • build.sh 对项目进行编译。

  • src 下不直接放置代码文件，而是将其归类存放在文件夹中。 

    test
├── bin
│   └── hello
├── build.sh
├── pkg
└── src
    └── hello
        └── hello.go

    hello.go 

    package main

import (
    "fmt"

    "github.com/phenix3443/go_test_lib/stringutils"
)

func main() {
    str := "hello,世界"
    fmt.Println(stringutils.Reverse(str))
}

    buiild.sh 

    #!/bin/bash
# 项目构建脚本
project_name=hello

if [ ! -d "src/${project_name}" ]; then
    echo "项目代码必须放在 src/${project_name}下"
    exit 1
fi

script_dir=$(cd $(dirname "$0"); pwd)
project_dir=$(realpath ${script_dir})
export GOPATH=${GOPATH}:${project_dir} # 将当前目录添加到
# echo ${GOPATH}

go get -v -x ${project_name}/...
go install ${project_name}

用于将你的 Go 代码从旧的发行版迁移到最新的发行版，它主要负责简单的、重复的、枯燥无味的修改工作，如果像 API 等复杂的函数修改，工具则会给出文件名和代码行数的提示以便让开发人员快速定位并升级代码。Go 开发团队一般也使用这个工具升级 Go 内置工具以及 谷歌内部项目的代码。go fix 之所以能够正常工作是因为 Go 在标准库就提供生成抽象语法树和通过抽象语法树对代码进行还原的功能。该工具会尝试更新当前目录下的所有 Go 源文件，并在完成代码更新后在控制台输出相关的文件名称。

出现错误： 

Could not attach to pid 6814: set /proc/sys/kernel/yama/ptrace_scope to 0

原因： https://www.kernel.org/doc/Documentation/security/Yama.txt, you will need to set that to 0 in order to attach to a running process. 解决办法： 

echo 0 | sudo tee /proc/sys/kernel/yama/ptrace_scope
