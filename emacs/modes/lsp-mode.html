1 概述

lsp-mode 是实现 lsp 的客户端，旨在通过集成 Emacs 最流行的软件包（如 company，flycheck 和 projectile）来提供类似 IDE 的体验。

emacs-cquery，lsp-rust，lsp-haskell 等可以看做是 lsp-mode 的适配器，用来适配对应的 lsp-server。

lsp-mode 主要功能：

  • 代码诊断，使用 flycheck/lsp-ui 或内置的 flymake（Emacs>26）。
  • Code completion（代码补全），使用 company-lsp 或内置的 complete-at-point。
  • 悬停展示，使用 lsp-ui，在代码附近展示相关信息：如函数原型提示等。
  • Code actions（代码操作），使用 lsp-execute-code-action 或 lsp-ui。
  • Code outline（代码大纲），使用内置的 imenu 或 helm-imenu。
  • Code navigation（代码导航） ，使用内置外部 xref。
  • Code lens（参考/实现），该功能提供正在使用的源代码的元信息，例如单元测试、更改历史等。使用内置 xref。
  • Highlights（代码高亮）
  • Code Format（代码格式化）
  • 调试器，使用 dap-mode

lsp-mode 在 lsp-clients.el 中预定义了几个 lsp-server（typescript,go,vue），这些 lsp-server 不需要额外的依赖。

2 配置

2.1 新增语言支持

以下是注册新语言服务器所需的最小配置。注册时支持的其他设置请参阅 lsp-clients.el 的文档。 必须更新 lsp-language-id-configuration 以包含相应的 mode->language id，此例是 (python-mode . "python") ： 

(lsp-register-client
 (make-lsp-client :new-connection (lsp-stdio-connection "pyls")
                  :major-modes '(python-mode)
                  :server-id 'pyls))

3 使用

使用 M-x lsp 启动。如果没有为当前模式注册客户端或相应服务器不存在，将回显消息。此外，lsp-mode 将自动检测和配置 lsp-ui 和 company-lsp。要关闭该行为，可以将 lsp-auto-configure 设置为 nil
