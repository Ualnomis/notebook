1 概述

Resilio Sync（以前称为 BitTorrent Sync）是 Resilio，Inc.发布的免费，快速，p2p 文件共享和同步工具。它可用于 Linux，Mac，FreeBSD，Windows，Android，iOS 和 NAS 设备。

2 安装

Windows 和 Mac 使用软件包进行安装，Linux 安装指南 https://help.resilio.com/hc/en-us/articles/206178924

3 使用

3.1 ubuntu1

ubuntu 上 resilio 使用 web 作为 gui 界面，服务以守护程序运行。

启动 Resilio Sync： 

sudo systemctl start resilio-sync

设置开机启动： 

sudo systemctl enable resilio-sync

检查服务运行状态： 

systemctl status resilio-sync

运行后，Resilio Sync 守护程序将侦听 /etc/resilio-sync/config.json 配置文件中指定的 127.0.0.1:8888 。现在，可以在 Web 浏览器地址栏输入 https://127.0.0.1:8888 访问 Resilio Sync Web UI。

将被要求设置用户名和密码来保护 Web UI。

3.2 文件夹权限问题

可能会遇到以下错误消息。这是因为 Resilio Sync 以 rslsync 用户身份运行。 

Don’t have permissions to write to selected folder.

假设选择了主文件夹 /home/your-username/ 作为共享文件夹。要解决以上错误，需要做的就是使用以下命令将主文件夹的权限授予 rslsync 用户。 

sudo setfacl -R -m "u:rslsync:rwx" /home/your-username

上面的命令不会更改共享文件夹的所有者。所有者拥有与往常相同的权限。它的作用是向另一个用户 rslsync 授予读取，写入和执行权限。请注意，-R（递归）标志必须在-m（修改）标志之前，然后紧随其后的是访问控制列表条目（u：rslsync：rwx）。

如果看到以下错误： 

sudo: setfacl: command not found

然后安装 acl 软件包并重新运行上面的 setfacl 命令。 

sudo apt install acl

3.3 作为服务端安配置

nginx 配置文件：resilio.proxy.nginx.conf 

server {
    listen 80;
    server_name resilio.your-domain;

    access_log /var/log/nginx/resilio.access.log;
    error_log /var/log/nginx/resilio.error.log;
    location / {
        proxy_pass http://127.0.0.1:8888;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

设置 resilio web gui 密码：/etc/resilio-sync/config.json 

{
    "storage_path" : "/var/lib/resilio-sync/",
    "pid_file" : "/var/run/resilio-sync/sync.pid",

    "webui" :
    {
        "listen" : "127.0.0.1:8888",
        "login":"username",
        "password":"password"
    }
}

然后就可以通过 http://resilio.your-domain/gui 进行访问了，要注意，此时 只能通过 http 进行访问 。

3.3.1 配置 https 进行访问

给 resilio.your-domain 申请好阿里云免费的 HTTPs 证书后，修改 resilio.proxy.nginx.conf 文件。 

server {
    listen 443 ssl;
    server_name resilio.your-domain;

    ssl_certificate /home/phenix/nginx/ssl/resilio.your-domain.pem;   #将domain name.pem替换成您证书的文件名。
    ssl_certificate_key /home/phenix/nginx/ssl/resilio.your-domain.key;   #将domain name.key替换成您证书的密钥文件名。
    ssl_session_timeout 5m;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4;  #使用此加密套件。
    ssl_protocols TLSv1 TLSv1.1 TLSv1.2;   #使用该协议进行配置。
    ssl_prefer_server_ciphers on;

    access_log /var/log/nginx/resilio.access.log;
    error_log /var/log/nginx/resilio.error.log;

    location / {
        proxy_pass http://127.0.0.1:8888$request_uri;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

server {
    listen 80;                  # 重定向HTTP请求为HTTPs
    server_name resilio.your-domain;
    return 301 https://resilio.your-domain$request_uri;
}

现在可以通过 http(s)://resilio.your-domain 直接访问 resilio web gui 域名
